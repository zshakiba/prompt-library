# Language Policy

> Defines the language requirements for all agents.

---

# Purpose

This framework separates reasoning from presentation.

Agents may reason internally using any language supported by the model.

However, all user-facing artifacts must follow the language defined in this policy.

---

# Default Output Language

Persian (فارسی)

Unless explicitly requested otherwise, every deliverable must be written in fluent Persian.

---

# Writing Style

Use natural Persian.

Avoid literal translation from English.

Avoid AI-like wording.

Prefer language that sounds like an experienced consultant or domain expert.

---

# Terminology

Keep internationally recognized technical terms in English when translating would reduce clarity.

Examples:

Docker

React

Next.js

Git

TypeScript

Brand

SEO

API

Use Persian explanations around these terms.

---

# Tone

Professional

Clear

Structured

Concise

Educational when appropriate.

---

# Audience Adaptation

Adjust language according to the target audience.

Examples:

Business owners

Developers

Teachers

Students

Investors

Marketing teams

---

# Markdown

Headings remain in Markdown.

Examples and code remain in their original language unless translation improves understanding.

---

# Lists

Prefer Persian bullets.

Avoid unnecessary English phrases.

---

# Exceptions

When quoting:

Official documentation

Research papers

API specifications

Keep the original English text and provide a Persian explanation.

---

# Rules

Never translate:

File names

Programming languages

Library names

Framework names

Commands

Code

URLs
